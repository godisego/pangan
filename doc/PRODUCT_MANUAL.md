# 📊 盘感 (Pangang) - AI 智能投研助手

## 1. 产品概述
**“盘感”** 是一款专为中国A股和加密货币投资者打造的现代化智能投研分析平台。它结合了实时市场数据、量化策略信号和生成式 AI 技术，旨在帮助投资者快速捕捉市场热点，辅助投资决策。

产品核心理念：**实时感知，量价为王，AI 赋能。**

---

## 2. 核心功能

### 2.1 📈 全景市场仪表盘 (Dashboard)
首页提供一站式的市场环境概览，帮助用户秒懂“当前是否适合操作”。

*   **A股大盘监控 (3s 刷新)**:
    *   实时显示上证指数、涨跌幅。
    *   关键指标：**涨跌家数比**（市场情绪温度计）、**涨停家数**（游资活跃度）。
    *   **北向资金**流向实时监控。
    *   **交易建议**: 系统根据量能和指数状态，自动给出“环境向好/风险加剧/空仓观望”的操作建议。
*   **加密货币行情 (10s 刷新)**:
    *   Bitcoin (BTC) 实时价格与24小时涨跌幅。
    *   **恐慌贪婪指数**: 实时追踪市场情绪（Fear & Greed Index）。
    *   关键信号标签：如“ETF持续流入”、“RSI偏强”等。

### 2.2 🔥 “量价齐升”智能选股
基于经典的量价分析理论，系统自动扫描全市场行业板块，寻找主升浪机会。

*   **策略逻辑**:
    *   **量价齐升**: 筛选涨幅 > 2% 且 换手率 > 3% 的活跃板块，捕捉资金合力方向。
    *   **动态分层**: 自动识别“领涨板块”（主攻）和“跟涨/滞涨板块”（补涨/防守）。
*   **稳健回退机制**:
    *   数据源采用**新浪财经 API**作为首选，确保高可用性。
    *   自动故障转移：若主数据源异常，自动切换至备用接口或显示历史数据，保障前端不白屏。

### 2.3 ⛓️ 产业链深度透视
点击任意热门板块，即可进入深度详情页。

*   **产业链结构化**:
    *   **核心领涨 (资金主攻)**: 当前市场最强个股，适合激进型选手。
    *   **虽迟但到 (板块联动)**: 尚未大涨但有补涨预期的个股，适合稳健型潜伏。
    *   **防守/滞涨**: 安全边际较高的防守型标的。
*   **个股指标**:
    *   实时显示成分股的价格、涨幅、换手率。
    *   **核心度评分**: 独家算法计算个股在该板块内的核心地位。

### 2.4 💬 AI 投研顾问
集成 Google Gemini 大模型的智能对话助手。

*   **交互式问答**: 用户可以问“今天为什么跌？”、“新能源板块还能追吗？”，AI 将结合市场概况给出分析。
*   **快捷指令**: 内置“今日复盘”、“热点解读”等快捷 Prompt，一键生成研报。

### 2.5 ⚡ 自动化预警与日报
*   **飞书 (Feishu) 集成**: 支持配置 Webhook 机器人。
*   **定时日报**: 每日早晨 08:00 自动推送市场前瞻与隔夜外盘总结，不错过任何重要信息。

## 3. 技术架构

本项目采用典型的前后端分离现代化架构：

*   **前端**:
    *   框架: **Next.js 16** (React 19)
    *   样式: **Tailwind CSS v4** (极致的深色模式 UI)
    *   图表: Lightweight Charts (高性能 K 线图)
*   **后端**:
    *   框架: **FastAPI** (Python)
    *   数据源: **新浪财经** (首选) + AKShare + 东方财富 (备选)
    *   AI 引擎: Google Generative AI (Gemini Pro)
    *   任务调度: APScheduler (定时任务)

## 4. 快速开始与配置

### 4.1 环境要求
*   Node.js > 18.0
*   Python > 3.9

### 4.2 启动服务
1.  **启动后端**:
    ```bash
    cd pangang-backend
    source venv/bin/activate
    python -m uvicorn app.main:app --reload
    ```
2.  **启动前端**:
    ```bash
    cd pangang
    npm run dev
    ```
    访问: `http://localhost:3000`

### 4.3 配置通知
1.  在飞书群组中添加“自定义机器人”，获取 Webhook 地址。
2.  进入应用 **设置** 页面。
3.  输入 Webhook URL 并保存。
4.  点击“测试发送”确认消息能否送达。

## 5. API 接口情况

后端服务运行在 `http://localhost:8000`，主要包含以下模块接口。

### 5.1 股票市场 (Stock Market)
核心数据模块，支持多源自动回退（Sina/EastMoney/AKShare）。

- `GET /api/stock/market`: 获取 A 股大盘实时环境（指数、北向资金、市场情绪）。
- `GET /api/stock/selection`: 获取**“量价齐升”**策略选股结果。
- `GET /api/stock/chain/{name}`: 获取指定概念板块的深度产业链成分股。
- `GET /api/stock/quotes`: 批量获取股票实时行情。
- `GET /api/stock/{code}/technical`: 获取个股技术分析指标（均线系统、K线形态）。
- `GET /api/stock/{code}/financial`: 获取个股财务摘要（营收、净利润、ROE）。
- `GET /api/stock/{code}/detail`: 获取个股基础详情。

### 5.2 加密货币 (Crypto)
目前主要基于 CoinGecko 和 OKX 公开接口。

- `GET /api/btc/summary`: 获取 BTC 实时摘要与恐贪指数。
- `GET /api/btc/technical`: 获取 BTC K 线与技术指标。
- `GET /api/btc/derivatives`: (Beta) 获取合约市场数据（资金费率）。
- `GET /api/btc/network`: (Beta) 获取链上健康度指标（算力、难度）。

### 5.3 通知服务 (Notification)
负责系统消息推送。

- `GET /api/notify/config`: 获取当前配置的 Webhook（脱敏）。
- `POST /api/notify/config`: 设置飞书 Webhook URL（持久化至 `config.json`）。
- `POST /api/notify/test`: 发送测试消息。
- `POST /api/notify/daily_report`: 手动触发日报推送任务。

## 6. 待完善与优化建议 (Roadmap)

### 6.1 数据源稳定性
*   **当前缺陷**: 系统严重依赖第三方免费/公开 API（新浪财经、东方财富、AKShare）。虽然实现了 API 故障转移机制，但仍存在接口变动、反爬虫限制导致服务不稳定的风险。
*   **优化建议**: 建议接入付费的专业金融数据服务（如聚宽、Tushare Pro），或建立本地数据清洗与缓存代理层，降低对单一直连接口的依赖。

### 6.2 历史数据与回测
*   **当前缺陷**: 当前架构为“实时流”模式，缺乏历史行情数据库。无法进行长周期的策略回测，也无法查看历史的选股准确率。
*   **优化建议**: 引入时序数据库（如 TimeScaleDB 或 ClickHouse）存储历史 K 线和每日选股结果，支持策略回溯验证。

### 6.3 账户体系与鉴权
*   **当前缺陷**: 系统目前为单机单用户设计，后端 API 没有任何鉴权机制（无 Token/Auth），Web 前端缺少用户登录系统。**请勿直接部署在公网环境**。
*   **优化建议**: 增加 JWT 身份认证，实现多用户独立的自选股管理、通知配置和 API 访问控制。

### 6.4 智能化深度 (RAG)
*   **当前缺陷**: 目前的 AI 分析主要基于预设 Prompt 和简单的行情数值，缺乏对实时新闻、公告、研报的语义理解。
*   **优化建议**: 引入 RAG（检索增强生成）技术，整合财经新闻源和公告数据库，让 AI 能真正解读“为什么涨”或“为什么跌”。

### 6.5 本地化与隐私
*   **当前缺陷**: Webhook 配置虽然保存在本地 JSON，但作为一个金融工具，缺乏加密存储。
*   **优化建议**: 对敏感配置（如 API Key、Webhook）进行加密存储。
